Web Application for Managing Physical Activity

Project Overview

The goal of this project is to develop a web application that enables users to track their physical activities, interact with others through posts and comments, and analyze their progress using charts. The platform emphasizes user-friendly design, data security, and social engagement. Built with Django and deployed on Heroku, the application integrates PostgreSQL for database management and Bootstrap for responsive UI design.

Key Features

The application offers a comprehensive set of features designed to promote physical activity and user engagement:

  • User registration and authentication (including Google OAuth 2.0)
  • Activity tracking with duration and category management
  • Social features: posts, comments, and user following
  • Data visualization through interactive bar and pie charts
  • CSV import/export for activity data
  • Privacy settings for user profiles
  • Admin dashboard for content moderation

Core Functionality

The application implements three main functional modules with detailed validation and security measures.

User Authentication

Implemented using Django Allauth with Google OAuth 2.0 integration. Features include:

  • Password strength validation
  • CSRF token protection
  • Profile privacy settings


Google OAuth Configuration

SOCIALACCOUNT_PROVIDERS = {
'google': {
'SCOPE': ['profile', 'email'],
'AUTH_PARAMS': {'access_type': 'online'}
}
}
Activity Management

Users can track physical activities with duration validation (5min-12hrs). Features include:

  • CSV import/export
  • Interactive charts using Plotly
  • Automatic time formatting


Activity Model

class Activity(models.Model):
user = models.ForeignKey(User, on_delete=models.CASCADE)
name = models.CharField(max_length=50)
duration = models.DurationField()
created_at = models.DateTimeField(auto_now_add=True)
